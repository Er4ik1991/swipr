# Swipr → Claude Code: инструкция и промпты по частям

Этот файл — пошаговая выдача задач для Claude Code. Копируешь **один блок за раз**, ждёшь
результат, проверяешь по «✅ Как проверить», и только потом даёшь следующий.

---

## ЧАСТЬ A. Как работать (прочитай один раз)

1. **Положи в папку проекта два файла:** твой прототип (далее `index.html` — замени на реальное имя)
   и `swipr_claude_code_brief.md`. Открой Claude Code в этой папке.
2. **Правило трёх единиц:** один промпт = один кусок работы = одна проверка. Не вставляй весь бриф
   разом — модель «размажет» внимание и наделает лишнего.
3. **Между шагами всегда проверяй** по блоку «✅ Как проверить». Не переходи к следующему промпту,
   пока текущий не проходит.
4. **Делай бэкап/коммит после каждой фазы** (`git commit` или просто копия файла). Если шаг сломал
   прототип — откатываешься на последнюю рабочую версию, а не чинишь поверх новыми фичами.
5. **Если что-то сломалось** — не пиши «доделай», а используй *шаблон-починку* из Части F.
6. **Бриф — источник истины.** В каждом промпте есть ссылка на его раздел; пусть Claude Code сверяется
   с ним, а не выдумывает.

## ЧАСТЬ C. Фаза 1 — вкладка «Чаты» и стейт-машина мэтчей

> Цель фазы: взаимный свайп создаёт мэтч → он виден во вкладке «Чаты» → первое сообщение переводит
> его в «Общаемся». Чат пока минимальный (только текст), приглашения — в Фазе 2.

### Промпт 1.1 — Навигация и каркас экрана чата
```
Бриф: разделы 1 и 7. Файл: index.html, остаётся single-file.

Задача:
1. Переименуй нижнюю вкладку «Отклики» → «Чаты» во ВСЕХ местах таббара (и на экране ленты/свайпа,
   и на самом экране). Обнови иконку при необходимости.
2. Создай новый пустой экран чата с id="chat": шапка (аватар-заглушка, имя, кнопка «назад»),
   пустая лента сообщений, поле ввода снизу. Пока без логики.
3. Настрой навигацию: go('chats') открывает список чатов, go('chat') открывает конкретный тред
   (пока заглушка). Кнопка «назад» в чате возвращает на go('chats').

Ничего из существующего не ломай: свайпы, DATA, go(), очередь, модалка мэтча должны работать как раньше.
В конце ответа: 1) что именно изменил, 2) как проверить вручную.
```
✅ **Как проверить:** вкладка называется «Чаты», открывается; есть пустой экран чата с «назад»;
свайпы и модалка мэтча работают по-прежнему.

### Промпт 1.2 — Стейт `negotiations` и создание треда при мэтче
```
Бриф: разделы 2 и 4.

Задача:
1. Заведи в стейте массив negotiations и структуру Negotiation по брифу (id, initiator,
   seekerId/employerId, vacancyId/resumeId, state, unread, lastMessageAt, hidden, messages[]).
2. При взаимном свайпе (мэтче) создавай тред в состоянии 'matched' и добавляй в negotiations.
3. Мигрируй существующий matches[] в negotiations[]: источник истины теперь negotiations
   (старый matches можно оставить как тонкий алиас, чтобы ничего не упало).
4. Счётчики navCount/navCount2 завяжи на сумму unread по всем тредам.

Логику колоды и свайпов не меняй сверх необходимого. В конце: что изменил + как проверить.
```
✅ **Как проверить:** в консоли `negotiations` растёт при каждом мэтче; новый тред имеет
`state:'matched'`; счётчик на иконке таба меняется.

### Промпт 1.3 — UI вкладки «Чаты»
```
Бриф: раздел 1.

Свёрстай содержимое вкладки «Чаты» одним проходом по negotiations:
1. Сверху — горизонтальный ряд кружков-аватаров «Новые мэтчи» = треды в состоянии 'matched'.
2. Ниже — вертикальный список диалогов = треды в состояниях talking/interview_invited/interview_set/offer:
   аватар, имя+должность (для соискателя) или вакансия+компания (для работодателя), текст последнего
   сообщения, время, бейдж непрочитанных, статус-чип состояния (Общаемся / Приглашение на собеседование /
   Оффер).
3. Треды rejected/hidden — вниз списка или скрой.
4. Тап по кружку или строке → go('chat') с этим тредом (передай id треда).

Стиль — как в текущем Swipr (те же шрифты Unbounded/Manrope и палитра). В конце: что изменил + как проверить.
```
✅ **Как проверить:** свежие мэтчи — кружками сверху; после переписки тред уезжает в список диалогов;
статус-чип и бейдж непрочитанных отображаются; тап открывает чат нужного треда.

### Промпт 1.4 — Минимальный чат и переход matched → talking
```
Бриф: раздел 3 (минимально) и раздел 4.

1. На экране чата выведи ленту messages выбранного треда: пузыри слева/справа, время.
2. Поле ввода + отправка текста: добавляет Message в тред, обновляет lastMessageAt.
3. Если тред был в состоянии 'matched' — первое сообщение переводит его в 'talking' и добавляет
   системное сообщение (по центру) о начале общения (Message.systemState = 'talking').
4. Открытие треда обнуляет его unread.
Пока БЕЗ вложений и БЕЗ приглашения на собеседование — только текст.

В конце: что изменил + как проверить, и прогони критерий приёмки Фазы 1 из брифа.
```
✅ **Как проверить (приёмка Фазы 1):** взаимный свайп → мэтч в «Новых мэтчах» → первое сообщение
переводит в «Общаемся» и двигает тред в список диалогов; счётчик непрочитанных корректен.

> **Здесь сделай коммит/бэкап.** Фаза 1 самостоятельна и уже даёт рабочий результат.

## ЧАСТЬ D. Фаза 2 — экран чата и приглашение на собеседование

### Промпт 2.1 — Полноценный экран чата
```
Бриф: раздел 3.

Дополни экран чата:
1. Системные сообщения по центру при смене состояния (отличаются от обычных пузырей).
2. 2–3 шаблона быстрых ответов («Расскажите подробнее», «Когда удобно созвониться?») — вставляют текст в поле.
3. Кнопка вложения (документ/файл) — пока заглушка, показывает «вложение прикреплено».
4. Меню треда (три точки в шапке): Скрыть тред (state='hidden'), Отклонить (state='rejected'),
   Пожаловаться (заглушка).
Не ломай отправку текста и переходы из Фазы 1. В конце: что изменил + как проверить.
```
✅ **Как проверить:** шаблоны подставляют текст; «Скрыть» убирает тред из списка; «Отклонить» уводит
его в rejected; системные сообщения видны по центру.

### Промпт 2.2 — Приглашение на собеседование (полное, с видеозвонком)
```
Бриф: разделы 3 и 4. Решение зафиксировано: приглашение делаем полноценным.

1. Кнопка/действие «Пригласить на собеседование» открывает форму:
   - формат: Видеозвонок / Офис / Телефон;
   - дата и время;
   - если Видеозвонок — поле ссылки (можно вставить свою или сгенерировать заглушку-ссылку);
   - если Офис — поле адреса/места.
2. После отправки в тред падает карточка-приглашение (Message.interview), состояние треда →
   'interview_invited', плюс системное сообщение.
3. На карточке у получателя — кнопки: Принять / Предложить другое время / Отклонить:
   - Принять → state='interview_set', статус карточки accepted;
   - Предложить другое время → state='talking' (возврат к переписке);
   - Отклонить → state='rejected'.
4. Для формата Видеозвонок в карточке показывай кнопку «Присоединиться» по ссылке; для Офиса — адрес.

В конце: что изменил + как проверить.
```
✅ **Как проверить:** можно создать приглашение всех трёх форматов; видео даёт кнопку «Присоединиться»,
офис — адрес; «Принять» меняет статус-чип на «Приглашение на собеседование»/interview_set; ветки
«Другое время» и «Отклонить» работают.

### Промпт 2.3 — Оффер и финал воронки
```
Бриф: раздел 4.

Добавь действие работодателя «Сделать оффер»: доступно в состоянии interview_set, переводит тред
в 'offer' с системным сообщением и статус-чипом «Оффер». Проверь, что проходит весь путь:
matched → talking → interview_invited → interview_set → offer, и обратные ветки (другое время, отказ).
В конце: что изменил + как проверить, прогони критерий приёмки Фазы 2 из брифа.
```
✅ **Как проверить (приёмка Фазы 2):** весь путь состояний проходится от начала до оффера и по обратным
веткам; системные сообщения и статус-чипы везде соответствуют состоянию.
